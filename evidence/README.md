# Runtime Evidence Directory

This directory separates **source readiness** from **runtime proof**.

- `baseline_normal/real_swarm_excerpt.log` contains selected evidence from successful candidate-machine runs used to harden the final revision.
- `runtime/` is created by `scripts/run_fault_campaign.sh` and contains one machine-readable JSON plus one reviewer-friendly Markdown report per mandatory fault.
- `RUNTIME_EVIDENCE_INDEX.md` is generated by `scripts/collect_submission_evidence.sh`.

Before submission, record one fresh normal mission using the packaged revision, run all five fault scenarios, and retain only the final passing evidence. Never manually change a `PASS` marker: it is written only when the scenario runner observes the required ROS messages and recovery conditions.
